# Non-Empty Multi-Row Implementation Plan (March 6, 2026)

## Goal
Integrate deterministic non-empty wire-topology synthesis into production codec for the validated lane:
- horizontal continuity (`-`, `T`)
- vertical continuity (`|`, `T`)
- tested scale: rows `2..32`

No instruction-stream family expansion in this pass.

## Proven Inputs (From RE)
- Vertical continuity authority: per-cell `+0x21` (down-link to next row).
- Horizontal continuity authority: per-cell `+0x19/+0x1D`, with observed asymmetry:
  - retaining `+0x1D` is compatible with preserving mixed `T` behavior
  - retaining only `+0x19` can collapse `T` to `|`
- Column scaling is direct (tested `c1` and `c3`).
- Row scaling validated through row32 (`row_word` and payload length preserved under verify-back).

## Scope
Include:
- wire-topology-only non-empty multi-row synthesis
- rows `2..32` logical rows
- token support in synthesis path: `""`, `-`, `|`, `T`

Exclude (follow-up):
- instruction-stream-heavy mixed families
- contact/coil stream placement changes
- speculative non-validated control bytes beyond proven wire flags

## Proposed Code Changes
1. New module for non-empty multi-row wire synthesis (parallel to `empty_multirow.py`).
2. Deterministic builder that:
   - allocates payload by validated row-length formula
   - writes row-word at header entry0 (`+0x00/+0x01`)
   - initializes structural baseline from scaffold/empty helper
   - applies wire flags per token to active cells:
     - `-` -> set `+0x19=1`, `+0x1D=1`, `+0x21=0`
     - `|` -> set `+0x19=0`, `+0x1D=0`, `+0x21=1`
     - `T` -> set `+0x19=1`, `+0x1D=1`, `+0x21=1`
3. Add guarded compatibility for horizontal asymmetry findings:
   - avoid emitting one-sided `+0x19`-only mixed cells in generated output.
4. Add explicit validation for `|` placement in column A (reject or normalize to blank).
5. Wire into codec only for explicitly selected multi-row non-empty wire path.

## Tests
Unit:
- row-word and payload length for rows `2,3,4,9,17,32`
- token-to-byte mapping for `-`, `|`, `T`
- invalid-token and column-A vertical guards

Fixture/roundtrip:
- promote representative passing scale captures (`gnms*`) and rowcombo captures (`gnmr*`)
- verify topology decode equality between generated bytes and expected token rows
- regression for asymmetry case:
  - generated `T` must not degrade to `|` in known-pass patterns

Manual verification gate:
- rerun curated queue with implementation output as file-backed entries before merge.

## Rollout Strategy
1. Implement synthesis module + tests (no codec wiring).
2. Add feature-guarded codec path and keep default behavior unchanged.
3. Run manual verify queue against generated payloads.
4. Remove guard/default once pass criteria are met.

## Open Risks
- asymmetry (`+0x19` vs `+0x1D`) may vary in instruction-heavy contexts not yet tested.
- session-seed/header-family interactions outside wire-only lane may still affect acceptance.
- need at least one post-integration manual verify sweep at rows `9/17/32`.
